# SEO Blog Article Generation Workflow

This n8n workflow automates the process of generating SEO-friendly blog articles, selecting relevant featured images from Pexels, uploading drafts to WordPress, and updating Google Sheets with the final publishing details.

The workflow is designed for teams that manage multiple websites and want to reduce manual work in blog writing, image selection, WordPress uploading, and tracking.

## Overview

This workflow uses AI agents, Google Sheets, Pexels, WordPress, and Gmail to create an end-to-end blog generation pipeline.

It performs the following tasks:

- Reads blog post requirements from Google Sheets
- Processes only rows where `done = false` or the status is unchecked
- Generates SEO-friendly blog content using the SEO Blog AI Agent
- Creates an image search query for the article
- Fetches 10 relevant landscape images from Pexels
- Uses the Image AI Agent to select the best featured image
- Generates image alt text
- Updates Google Sheets with image details
- Downloads the selected Pexels image
- Routes the article to the correct WordPress website
- Uploads the featured image to the WordPress media library
- Updates the uploaded image alt text
- Creates the blog post as a WordPress draft
- Sets the featured image for the draft post
- Updates Google Sheets with the WordPress preview URL
- Marks the row as completed
- Sends a completion email
- Logs success and error runs into the spreadsheet

## How It Works

### 1. Read Blog Information from Google Sheets

The workflow starts by getting all required blog post information from Google Sheets.

Each row should include details such as:

- Website
- Language
- Word count
- Meta title
- Meta description
- Primary keyword
- Target/internal URL
- Completion status

Only rows where `done = false` or the checkbox is unchecked will be processed.

---

### 2. Process Each Unchecked Row

The workflow loops through all unchecked rows one by one.

For each row, it passes the blog details to the SEO Blog AI Agent.

---

### 3. Generate SEO Blog Content

The SEO Blog AI Agent creates a complete SEO-friendly blog draft based on the spreadsheet input.

The generated output includes:

- Website
- SEO meta title
- SEO meta description
- Blog title / H1
- WordPress-ready HTML blog content
- Suggested URL slug
- Primary keyword
- Pexels image search query
- Internal link used in the article

The blog content is returned in clean HTML format, using tags such as:

- `<h2>`
- `<h3>`
- `<p>`
- `<ul>`
- `<li>`
- `<a>`

The H1 is returned separately as the WordPress post title.

---

### 4. Fetch Images from Pexels

The workflow uses the `image_search_query` generated by the SEO Blog AI Agent to search Pexels.

It fetches 10 relevant landscape images using the Pexels API.

---

### 5. Select the Best Featured Image

The Image AI Agent reviews the Pexels results and selects the most suitable image for the blog post.

The selection is based on:

- Blog topic
- Primary keyword
- Search intent
- Meta description
- Website audience
- Image quality
- Featured image suitability

The Image AI Agent returns:

- Selected image URL
- Alt text
- Photographer name
- Pexels page URL
- Reason for selection

---

### 6. Update Image Data in Google Sheets

After the image is selected, the workflow updates the spreadsheet with:

- `photo_url`
- `alt_text`
- `photographer`
- `pexels_url`

---

### 7. Download the Selected Image

The selected image is downloaded from Pexels and prepared for upload to WordPress.

---

### 8. Route to the Correct WordPress Website

The workflow checks the `website` value from the spreadsheet and maps the blog post to the correct WordPress site.

Example supported websites:

- `istanabambu.com.my`
- `inimajalah.com.my`

If the website is not mapped, the workflow logs an error in the spreadsheet.

---

### 9. Upload Image to WordPress

The downloaded image is uploaded to the selected WordPress website’s media library.

After upload, the workflow updates the image alt text using the alt text generated by the Image AI Agent.

---

### 10. Create WordPress Draft Post

The workflow creates a new WordPress post as a draft.

The draft includes:

- Blog title
- Blog content
- Suggested URL slug
- Draft status
- Featured image

The workflow does not publish the article automatically. This allows users to review the draft before publishing.

---

### 11. Set Featured Image

Once the draft post is created, the workflow sets the uploaded media item as the featured image for the post.

---

### 12. Update Google Sheets

After the WordPress draft is created, the workflow updates Google Sheets with:

- `wp_url`
- `done = true`

This allows users to preview the generated WordPress draft directly from the spreadsheet.

---

### 13. Send Completion Email

When all rows have been processed, the workflow sends a completion email to notify the admin or user that the blog drafts are ready for review.

---

### 14. Log Workflow Results

The workflow logs both successful and failed runs into the spreadsheet.

Logs include:

- Reference number
- Website
- Status
- Message
- Timestamp

---

## Google Sheets Setup

Use the provided spreadsheet template and fill in the required fields.

Required input columns include:

| Column | Description |
|---|---|
| `no` | Unique row reference number |
| `website` | Target WordPress website |
| `language` | Blog language |
| `word_count` | Target word count |
| `meta_title` | SEO meta title |
| `meta_desc` | SEO meta description |
| `keyword` | Primary SEO keyword |
| `targeted_url` | Internal link to include in the blog |
| `done` | Processing status |

The workflow will update these columns automatically:

| Column | Description |
|---|---|
| `photo_url` | Selected Pexels image URL |
| `alt_text` | Generated image alt text |
| `photographer` | Pexels photographer name |
| `pexels_url` | Original Pexels image page |
| `wp_url` | WordPress draft preview URL |
| `done` | Updated to `true` after successful processing |

## Required Credentials

Before running the workflow, configure the following credentials in n8n.

### Google Sheets

Used to read blog input rows and update workflow results.

Required for:

- Reading blog information
- Updating image data
- Updating WordPress URLs
- Logging success and error runs

---

### OpenAI

Used by:

- SEO Blog AI Agent
- Image AI Agent

The workflow currently uses `gpt-5-mini` for both agents.

---

### Pexels API

Used to fetch featured image options.

Setup steps:

1. Go to the Pexels API page.
2. Generate or copy your API key.
3. Add the API key as a header authentication credential in n8n.
4. Use `Authorization` as the header name.
5. Paste your Pexels API key as the value.

---

### WordPress

Used to create draft posts and update post data.

Each WordPress website requires its own credentials.

You will need:

- WordPress username
- WordPress application password
- Basic Auth credential for media upload and featured image update
- WordPress API credential for post creation

---

### Gmail

Used to send the final completion email.

Configure Gmail OAuth credentials in n8n before enabling the notification step.

---

## Adding More WordPress Websites

To add another WordPress website:

1. Copy an existing website node group.
2. Update the WordPress credentials.
3. Update the Basic Auth credentials.
4. Edit the `Map to WordPress Site` node.
5. Add a new routing rule using the new website domain.
6. Connect the new route to the copied website node group.
7. Connect all error outputs to the HTTP error logging flow.

Example routing rule:

```text
$json.website equals yourdomain.com
```

---

## Error Handling

The workflow includes error handling for:

- Pexels image fetching errors
- Image download errors
- WordPress media upload errors
- WordPress post creation errors
- Featured image update errors

Errors are logged into the Logs sheet with a failure message and timestamp.

---

## Empty Run Handling

If all rows are already marked as completed, the workflow logs an empty run with the message:

```text
All rows are checked. Nothing to process.
```

---

## Final Output

After a successful run, each processed row will have:

- A generated WordPress draft post
- A selected featured image
- Updated image attribution details
- A WordPress preview URL
- done marked as true
- A success log in the spreadsheet

The admin or user will also receive a completion email confirming that the blog drafts are ready for review.

---

## Notes

- The workflow creates posts as drafts only.
- Users should review all generated posts before publishing.
- The workflow does not use photo_url, docs_url, done, or wp_url when generating blog content.
- Pexels image attribution details are stored in the spreadsheet for reference.
- Website routing must be configured correctly before processing rows for new domains.
